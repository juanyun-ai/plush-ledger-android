# 更新日志

## 0.9.7 - 2026-06-21

1. 修复旧下载记录在每次打开 APP 时自动弹出“正在下载”的问题。
2. 更新弹窗增加真实下载状态：等待启动、已下载字节、百分比、安全校验和明确失败原因。
3. 下载期间不再重复显示“发现新版本”弹窗，也不会因重复点击创建冲突任务。
4. 增加取消、断点续传和失败后重新下载入口；只有收到真实文件字节才显示下载百分比。
5. 0.9.6 国内镜像已补传到 Supabase Storage，并通过文件大小与 SHA-256 双重校验。

## 0.9.6 - 2026-06-21

1. 修复点击下载后没有真实下载任务的问题，更新包现在由系统后台任务持续下载。
2. 增加系统通知下载进度、失败原因、断点续传、双线路切换和下载完成安装入口。
3. 修复“国内优先”设置未生效，Supabase 下载源会优先于 GitHub 使用。
4. 下载完成后继续执行 SHA-256 完整性校验，不完整或被篡改的安装包不会进入安装流程。

## 0.9.5 - 2026-06-21

1. 邮箱、手机、本地登录界面及区号弹窗按最新产品设计稿重构。
2. QQ 登录标志更新为企鹅形态，手机号页明确标注短信服务暂未开放。
3. AI 输入和确认弹窗按设计稿重构，新增识别动画并在重新打开时清空历史输入。
4. AI 解析优先服从用户明确指定的日期、分类和账户，支持具体日期、相对日期与跨月表达。
5. 修复日期数字被识别成金额及云端记录回写时覆盖原交易时间的问题。
6. 移除语音入口和麦克风权限，减少不必要的隐私授权。
7. “我的”新增宠物绒绒专题，支持互动、陪伴值、心情、每日照顾和奖励状态。

## 0.9.4 - 2026-06-21

1. 应用内更新改为自带下载器，避免部分设备的系统下载器在大文件下载时出现断流失败。
2. 更新包支持断点续传、主备线路自动切换和 SHA-256 完整性校验，只有校验成功才会进入系统安装页。
3. 已发布的 0.9.3 版本将 GitHub Release 调整为主下载线路，Supabase Storage 保留为备用线路，旧版本也会立即受益。

## 0.9.3 - 2026-06-21

1. 修复未选择分类时保存会清空金额和备注的问题，校验失败后继续保留用户输入。
2. AI 记账在确认时确保账本、账户和分类有效；保存失败会保留识别建议并提示原因。
3. AI 本地规则补全正餐、房租、人情红包、理财收益等表达，并消除“随礼红包”被误判为服饰鞋包的问题。
4. 官方信箱改为逐条容错和本地历史缓存，网络波动或单个版本信息异常不再只显示当天欢迎语。
5. 旧分类名称自动映射到新版毛绒图标，历史账目无需重新录入。
6. 语音输入增加设备语音服务检测，清晰区分麦克风权限与系统服务不可用。

## 0.9.2 - 2026-06-21
