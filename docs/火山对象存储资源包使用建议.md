## 适合用来做什么

对象存储适合存放文件，不适合作为记账数据库。

适合：

- Android APK 备用下载镜像。
- 官网图片、截图、宣传图、压缩包。
- 其他项目的静态文件、课程资料、作品集素材。
- 用户主动导出的备份文件，但必须走后端签名上传/下载，并设置生命周期清理。

谨慎：

- 用户头像、账单截图、附件。可以用，但必须私有桶、签名 URL、限制大小和类型。
- 小程序或 App 的公共资源。可以用，但注意公网流出流量。

不建议：

- 用户记账流水、账户资料、会话、支付信息。
- 任何需要复杂查询、权限关系、事务一致性的核心数据。
- 把 TOS AccessKey 写进 App、小程序或网页前端。

## 绒绒记账可用方案

优先级从高到低：

1. 做 APK 备用下载源
   - 继续以 GitHub Release 或 Supabase Storage 作为主下载。
   - 把 APK 同步一份到 TOS。
   - 在 `app_versions.backup_apk_url` 里填写 TOS 下载地址。
   - 这样正好消耗 10GiB 存储包和 10GB 公网流量包。
   - 前提：先开通 TOS，创建私有 bucket 或只读公共下载 bucket。

2. 做产品官网静态资源源
   - 官网仍部署在 Cloudflare Pages。
   - 大图、截图、素材可以放 TOS，再由官网引用。
   - 当前官网很轻量，不强求迁移。

3. 做手动备份仓库
   - 保存发布包、审核截图、备案材料、隐私协议 PDF。
   - 桶建议私有，只给管理员下载。

4. 给其他项目复用
   - 题海登科、小红书素材、课程资料、作品集下载包都可以复用同一个火山账号的 TOS。
   - 建议按项目建不同目录或不同桶，避免混在一起。

## 安全边界

- 桶默认私有。
- 不在客户端暴露 AccessKey。
- 上传、下载走后端签名，签名 URL 设置短有效期。
- 设置文件大小限制和 MIME 类型白名单。
- 定期清理临时文件。
- 用户隐私数据仍以 Supabase 数据库为主，依赖 RLS、Auth 和 Edge Function 控制权限。

## 是否值得续费

以当前页面价格看，约 4.45 元/月，不贵。只要你还会发布 APK、放产品素材、做其他项目文件托管，就可以续。

如果到期前 1 个月仍完全没用到，可以取消续费；如果已经把 APK 镜像或项目素材放进去，就续这个小规格即可，不需要升级大包。
